# 标准项目目录结构

## 项目根目录结构
```
jsp-scheduling-agent/
├── .trae/                     # Trae项目级配置（团队共享、提交Git）
│   ├── rules/                 # 项目开发规则，AI强制遵守
│   ├── mcp.json               # 项目MCP服务配置
│   ├── agents/                # 项目专用智能体配置
│   └── settings.json          # 项目级AI行为设置
├── data/                      # 输入数据层
│   ├── test_case.py           # 内置标准JSP测试数据集
│   └── sample_data.csv        # 外部导入数据示例
├── config/                    # 参数配置层
│   └── settings.py            # GA超参数、全局随机种子等配置
├── core/                      # 核心算法层
│   ├── jsp_model.py           # JSP模型实例化与约束校验
│   ├── decoder.py             # 贪婪解码器
│   ├── genetic_algorithm.py   # IPOX交叉+工件块移位变异GA
│   └── baseline_solvers.py    # FCFS/SPT基线调度算法
├── utils/                     # 工具层
│   ├── metrics_calculator.py  # KPI指标计算
│   └── visualizer.py          # 甘特图、收敛曲线绘制
├── main.py                    # 程序主入口
├── requirements.txt           # 依赖清单
└── docs/                      # 项目文档
    └── design_doc.md          # 终版设计文档
```

## 目录职责说明
- **.trae/**: 项目配置和规则目录，所有AI开发助手必须遵循的规范
- **data/**: 存放测试数据和样本数据，支持多种数据格式
- **config/**: 集中管理所有配置参数，避免硬编码
- **core/**: 核心算法实现，确保算法独立性和可测试性
- **utils/**: 工具函数和可视化组件，支持算法分析和展示
- **docs/**: 项目文档，包含设计文档、API文档等
