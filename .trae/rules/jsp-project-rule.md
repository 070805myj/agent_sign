# JSP项目开发规则

## 项目概述
本项目为作业车间调度优化智能体，采用遗传算法求解经典JSP问题，支持基线算法对比和可视化分析。

## 核心开发原则
1. **严格遵守规则**: 所有代码必须符合 `.trae/rules/` 目录下的所有规则
2. **设计文档导向**: 所有实现必须与 `docs/design_doc.md` 设计文档完全一致
3. **算法正确性优先**: 确保算法实现的数学正确性和工程正确性
4. **可复现性要求**: 使用固定随机种子，确保实验结果可重复

## 技术栈要求
- **编程语言**: Python 3.8+
- **核心依赖**: NumPy, Matplotlib, Pandas
- **算法框架**: 自研遗传算法（IPOX交叉 + 工件块移位变异）
- **测试框架**: unittest 或 pytest

## 开发流程
1. 设计文档评审和确认
2. 按照 `标准项目目录结构.md` 搭建项目框架
3. 逐模块开发并验证算法正确性
4. 集成测试和性能优化
5. 文档完善和交付

## 质量保证
- 代码审查必须覆盖所有核心算法实现
- 单元测试覆盖率不低于80%
- 必须通过标准测试用例验证
- 输出结果必须符合设计文档规范

## 版本控制
- 使用 Git 进行版本管理
- 提交信息必须清晰描述修改内容
- 重要修改需要更新相应文档
- 保护 `.trae/` 目录内容的完整性
