# 基于改进遗传算法的作业车间调度问题求解研究

## 摘要

作业车间调度问题（Job-Shop Scheduling Problem, JSP）是运筹学中经典的NP-hard问题，具有重要的理论意义和工程应用价值。本文针对JSP问题，设计并实现了一种基于改进遗传算法（Improved Genetic Algorithm, IGA）的求解框架。算法采用IPOX（Improved Precedence Preservative Crossover）交叉算子和工件块移位变异（Job Block Shift Mutation）算子，结合左移贪婪解码策略，有效平衡了全局探索与局部开发能力。同时，引入自适应变异率机制和精英保留策略，进一步提升了算法的收敛速度和解质量。为验证算法有效性，实现了FCFS（先来先服务）和SPT（最短加工时间优先）两种基线算法进行对比实验。实验结果表明，本文算法在标准测试用例和随机生成的大规模实例上均显著优于基线算法。

**关键词**：作业车间调度；遗传算法；IPOX交叉；贪婪解码；自适应变异

## 1 引言

### 1.1 研究背景

作业车间调度问题（JSP）研究如何在满足工艺约束和机器约束的前提下，合理安排工件在各机器上的加工顺序，以优化特定的调度目标（如最大完工时间）。JSP广泛存在于制造业、半导体生产、物流运输等领域，是智能制造系统中的核心优化问题。

JSP的求解难度极高。对于n个工件、m台机器的问题，解空间规模为 $(n!)^m$，当 $n=10, m=10$ 时，解空间约为 $3.6 \times 10^{65}$，远超现代计算机的枚举能力。因此，精确算法（如分支定界、整数规划）仅适用于小规模实例，大规模JSP通常采用元启发式算法求解。

### 1.2 研究内容

本文设计并实现了一个完整的JSP求解系统，主要贡献包括：

1. **改进遗传算法**：采用IPOX交叉算子保持工件间的相对顺序，使用工件块移位变异算子增强局部搜索能力，结合自适应变异率和精英保留策略
2. **左移贪婪解码器**：将工序序列编码的染色体转换为满足所有约束的调度方案，时间复杂度为 $O(n \cdot m)$
3. **基线对比算法**：实现FCFS和SPT两种经典调度规则作为对比基准
4. **完整评估体系**：包含Makespan、设备利用率、负载方差、优化率等多维度KPI指标

## 2 问题描述与数学模型

### 2.1 问题定义

设有 $n$ 个工件 $J = \{J_0, J_1, \ldots, J_{n-1}\}$ 和 $m$ 台机器 $M = \{M_0, M_1, \ldots, M_{m-1}\}$。每个工件 $J_j$ 包含 $n_j$ 道工序 $O_{j,0}, O_{j,1}, \ldots, O_{j,n_j-1}$，必须按给定顺序依次加工。每道工序 $O_{j,k}$ 在指定机器 $\mu_{j,k}$ 上加工，加工时间为 $p_{j,k}$。

**符号说明：**

| 符号 | 含义 |
|------|------|
| $n$ | 工件数量 |
| $m$ | 机器数量 |
| $J_j$ | 第 $j$ 个工件（$j = 0, 1, \ldots, n-1$） |
| $O_{j,k}$ | 工件 $J_j$ 的第 $k$ 道工序（$k = 0, 1, \ldots, n_j-1$） |
| $\mu_{j,k}$ | 工序 $O_{j,k}$ 的加工机器编号 |
| $p_{j,k}$ | 工序 $O_{j,k}$ 的加工时间（$p_{j,k} > 0$） |
| $s_{j,k}$ | 工序 $O_{j,k}$ 的开始时间 |
| $c_{j,k}$ | 工序 $O_{j,k}$ 的完工时间，$c_{j,k} = s_{j,k} + p_{j,k}$ |
| $C_{\max}$ | 最大完工时间（Makespan） |

### 2.2 约束条件

**约束1：工序顺序约束**

同一工件的工序必须按工艺顺序依次加工，前一道工序完成后才能开始下一道：

$$s_{j,k} \geq c_{j,k-1}, \quad \forall j \in \{0,\ldots,n-1\}, \forall k \in \{1,\ldots,n_j-1\}$$

即：

$$s_{j,k} \geq s_{j,k-1} + p_{j,k-1}$$

**约束2：机器独占约束**

同一机器同一时刻只能加工一道工序。设 $A_m$ 为机器 $m$ 上加工的所有工序集合，则对于任意两个工序 $O_{j,k}, O_{j',k'} \in A_m$，有：

$$s_{j,k} \geq c_{j',k'} \quad \text{或} \quad s_{j',k'} \geq c_{j,k}$$

**约束3：非负约束**

所有工序的开始时间和完工时间非负：

$$s_{j,k} \geq 0, \quad c_{j,k} \geq 0$$

### 2.3 目标函数

本文以最小化最大完工时间（Makespan）为目标：

$$\min C_{\max} = \min \max_{j \in \{0,\ldots,n-1\}} c_{j,n_j-1}$$

等价于：

$$\min C_{\max} = \min \max_{m \in \{0,\ldots,m-1\}} T_m$$

其中 $T_m$ 为机器 $m$ 的最后完工时间。

### 2.4 辅助指标

除Makespan外，本文还计算以下KPI指标用于综合评估调度方案质量：

**（1）总完工时间**

$$C_{\text{total}} = \sum_{j=0}^{n-1} c_{j,n_j-1}$$

**（2）平均流程时间**

由于标准JSP中所有工件在0时刻到达，平均流程时间为：

$$\bar{F} = \frac{1}{n} \sum_{j=0}^{n-1} c_{j,n_j-1} = \frac{C_{\text{total}}}{n}$$

**（3）机器平均利用率**

$$U = \frac{\sum_{j=0}^{n-1} \sum_{k=0}^{n_j-1} p_{j,k}}{m \cdot C_{\max}}$$

分子为所有工序的总加工时间，分母为机器总可用时间。

**（4）机器负载方差**

设 $W_m$ 为机器 $m$ 的总加工负载：

$$W_m = \sum_{\substack{j,k \\ \mu_{j,k}=m}} p_{j,k}$$

平均负载：

$$\bar{W} = \frac{1}{m} \sum_{m=0}^{m-1} W_m$$

负载方差：

$$\sigma^2 = \frac{1}{m} \sum_{m=0}^{m-1} (W_m - \bar{W})^2$$

**（5）总空闲时间**

$$I_{\text{total}} = m \cdot C_{\max} - \sum_{j=0}^{n-1} \sum_{k=0}^{n_j-1} p_{j,k}$$

**（6）优化率**

以基线算法Makespan为参照：

$$R_{\text{opt}} = \frac{C_{\max}^{\text{baseline}} - C_{\max}^{\text{GA}}}{C_{\max}^{\text{baseline}}} \times 100\%$$

## 3 遗传算法设计

### 3.1 染色体编码

本文采用**基于工序的编码**（Operation-based Encoding）。染色体为一个长度为 $N = \sum_{j=0}^{n-1} n_j$ 的序列 $\pi = (\pi_1, \pi_2, \ldots, \pi_N)$，其中 $\pi_i \in \{0, 1, \ldots, n-1\}$，工件编号 $j$ 在序列中出现 $n_j$ 次。

**解码规则**：按从左到右的顺序扫描染色体，第 $k$ 次出现的工件编号 $j$ 表示工件 $J_j$ 的第 $k-1$ 道工序。这种编码方式天然满足工序顺序约束，且任意排列都对应一个合法的调度方案。

**示例**：设 $n=3$，每个工件有3道工序，染色体 $\pi = (2, 0, 1, 2, 0, 1, 2, 0, 1)$ 解码为：
- $O_{2,0} \to O_{0,0} \to O_{1,0} \to O_{2,1} \to O_{0,1} \to O_{1,1} \to O_{2,2} \to O_{0,2} \to O_{1,2}$

### 3.2 种群初始化

采用随机初始化策略：

1. 为每个工件 $J_j$ 在序列中添加 $n_j$ 个 $j$
2. 对整个序列进行随机洗牌

该方式保证初始种群具有多样性，且每个个体均为合法解。

### 3.3 适应度函数

适应度函数采用Makespan的倒数：

$$f(\pi) = \frac{1}{C_{\max}(\pi)}$$

其中 $C_{\max}(\pi)$ 为染色体 $\pi$ 经左移贪婪解码后的最大完工时间。由于Makespan恒为正值，适应度值越大表示个体越优。采用倒数形式将最小化问题转化为最大化问题，便于选择操作。

**适应度缓存优化**：为避免对相同个体重复解码，采用哈希表缓存已计算个体的适应度值。以染色体的元组表示作为键，适应度值为值。实验表明，该优化可减少约26%的解码次数。

### 3.4 选择算子：截断选择

采用截断选择（Truncation Selection）策略：

1. 按适应度降序对种群排序
2. 选择前 $T$ 个个体作为父代池，其中：

$$T = \max\left(2, \left\lfloor \frac{N_{\text{pop}}}{S} \right\rfloor\right)$$

其中 $N_{\text{pop}}$ 为种群大小，$S$ 为选择压力（默认 $S = 2.0$）。

该策略简单高效，选择压力通过截断比例间接控制。较大的 $S$ 值意味着更强的选择压力和更小的父代池。

### 3.5 交叉算子：IPOX

IPOX（Improved Precedence Preservative Crossover）是一种保持工件间相对加工顺序的交叉算子，由Lee等人在研究JSP时提出。其核心思想是将工件随机划分为两个子集，子代分别继承两个父代在不同子集上的顺序信息。

**算法步骤：**

**输入**：父代 $P_1$, $P_2$

**步骤1**：将工件集合 $J$ 随机划分为两个互不相交的子集 $G_1$ 和 $G_2$，满足 $G_1 \cup G_2 = J$，$G_1 \cap G_2 = \emptyset$。

**步骤2**：生成子代 $C_1$：
- 扫描 $P_1$，保留 $P_1$ 中属于 $G_1$ 的基因位置
- 对 $P_1$ 中属于 $G_2$ 的位置，依次填入 $P_2$ 中属于 $G_2$ 的基因（保持 $P_2$ 中的相对顺序）

**步骤3**：生成子代 $C_2$：
- 扫描 $P_2$，保留 $P_2$ 中属于 $G_2$ 的基因位置
- 对 $P_2$ 中属于 $G_1$ 的位置，依次填入 $P_1$ 中属于 $G_1$ 的基因（保持 $P_1$ 中的相对顺序）

**输出**：子代 $C_1$, $C_2$

**数学正确性证明：**

IPOX保持工件工序计数不变。设工件 $j$ 在 $P_1$ 中出现 $n_j$ 次（即 $j \in G_1$ 或 $j \in G_2$）：
- 若 $j \in G_1$：$C_1$ 中 $j$ 来自 $P_1$，出现 $n_j$ 次
- 若 $j \in G_2$：$C_1$ 中 $j$ 来自 $P_2$，出现 $n_j$ 次

因此 $C_1$ 中每个工件 $j$ 恰好出现 $n_j$ 次，为合法染色体。同理可证 $C_2$ 合法。

**示例：**

设 $n=4$，每个工件2道工序，$P_1 = (1, 2, 3, 0, 1, 2, 3, 0)$，$P_2 = (0, 3, 2, 1, 0, 3, 2, 1)$。

随机划分 $G_1 = \{0, 1\}$, $G_2 = \{2, 3\}$。

生成 $C_1$（保留 $P_1$ 中 $G_1$ 的位置，填入 $P_2$ 中 $G_2$ 的基因）：
- $P_1$ 中 $G_1$ 位置（索引3,4,7,8，值为0,1,0,1）保留
- $P_2$ 中 $G_2$ 基因序列为 $(3, 2, 3, 2)$
- $C_1 = (3, 2, 3, 0, 1, 2, 3, 0)$... （按位置填充）

### 3.6 变异算子：工件块移位变异

工件块移位变异（Job Block Shift Mutation）通过将某个工件的所有工序作为整体进行移位，增强局部搜索能力。

**算法步骤：**

**输入**：染色体 $\pi$

**步骤1**：随机选择一个工件 $j \in \{0, 1, \ldots, n-1\}$

**步骤2**：从 $\pi$ 中提取所有值为 $j$ 的基因，形成工件块 $B_j$，剩余基因保持原相对顺序形成序列 $R$

**步骤3**：随机选择插入位置 $p \in \{0, 1, \ldots, |R|\}$

**步骤4**：将 $B_j$ 插入到 $R$ 的位置 $p$，形成新染色体 $\pi'$

$$\pi' = R[0:p] + B_j + R[p:|R|]$$

**输出**：变异后染色体 $\pi'$

**特点：** 该算子保持工件块内部的工序顺序不变（因为同一工件的所有工序值相同），仅改变工件间的相对顺序，属于保序变异。

### 3.7 自适应变异率机制

为平衡算法的探索（Exploration）与开发（Exploitation），设计自适应变异率调节机制。

**多样性度量：** 采用种群适应度标准差作为多样性指标：

$$\sigma_g = \sqrt{\frac{1}{N_{\text{pop}}} \sum_{i=1}^{N_{\text{pop}}} (f_i - \bar{f})^2}$$

其中 $f_i$ 为第 $i$ 个个体的适应度，$\bar{f}$ 为种群平均适应度。

**基线建立：** 前10代收集适应度标准差，取中位数作为基线 $\sigma_{\text{base}}$。

**触发条件：** 当满足以下任一条件时，提升变异率：
- 当前多样性低于阈值：$\sigma_g < \theta \cdot \sigma_{\text{base}}$（$\theta$ 为多样性比例阈值，默认0.2）
- 最优解连续停滞：连续 $N_{\text{early}}$ 代最优解无更新

**调节规则：**

$$p_m = \begin{cases} \min(0.5, 1.5 \cdot p_m) & \text{若触发条件满足} \\ \max(p_m^{\text{init}}, 0.99 \cdot p_m) & \text{若 } \sigma_g > 1.5 \theta \sigma_{\text{base}} \end{cases}$$

该机制在种群多样性下降时增强变异以注入新基因，在多样性恢复后逐步衰减变异率以加强收敛。

### 3.8 精英保留策略

为防止优秀个体在遗传操作中丢失，采用精英保留策略：

1. 按适应度降序排序当前种群
2. 选择前 $N_e$ 个个体直接复制到下一代：

$$N_e = \max\left(1, \lfloor \alpha \cdot N_{\text{pop}} \rfloor\right)$$

其中 $\alpha$ 为精英比例（默认0.05，即5%）。

3. 其余 $N_{\text{pop}} - N_e$ 个个体通过交叉和变异生成

精英保留保证每代最优解不劣于上一代，是遗传算法收敛性的重要保障。

### 3.9 早停机制

为避免不必要的计算，设置早停条件：若连续 $N_{\text{early}}$ 代（默认50代）最优Makespan无改进，则终止算法。

### 3.10 算法整体流程

**算法1：改进遗传算法求解JSP**

```
输入：JSP模型 Model, 参数 Config
输出：最优调度方案 Schedule*

1.  初始化种群 P(0) ← 随机生成 N_pop 个个体
2.  评估 P(0)，计算每个个体的适应度
3.  记录全局最优解 π* ← argmax f(π), g ← 0
4.  WHILE g < G_max DO
5.      更新自适应变异率 pm
6.      父代池 Q ← 截断选择(P(g))
7.      子代 O ← 对 Q 执行IPOX交叉（概率 pc）和工件块移位变异（概率 pm）
8.      精英 E ← 精英保留(P(g))
9.      P(g+1) ← E ∪ O
10.     评估 P(g+1)
11.     IF 当前最优 > 全局最优 THEN 更新 π*, 重置停滞计数
12.     ELSE 停滞计数 +1
13.     IF 停滞计数 ≥ N_early THEN BREAK
14.     g ← g + 1
15. END WHILE
16. Schedule* ← 贪婪解码(π*)
17. RETURN Schedule*
```

## 4 左移贪婪解码器

左移贪婪解码器（Left-shift Greedy Decoder）将染色体转换为具体的调度时间表，是算法的核心组件。

### 4.1 解码原理

对于给定的工序序列，按顺序为每道工序安排尽可能早的开始时间，即"左移"到最早可行位置。

### 4.2 解码算法

**算法2：左移贪婪解码**

**输入**：工序序列 $\pi$，JSP模型 $Model$

**输出**：调度方案（包含每道工序的开始时间、完工时间）

```
1.  初始化机器可用时间 M[i] ← 0, ∀i ∈ {0,...,m-1}
2.  初始化工件工序指针 k[j] ← 0, ∀j ∈ {0,...,n-1}
3.  初始化工件完工时间 C[j] ← 0, ∀j ∈ {0,...,n-1}
4.  FOR each gene π_i in π DO
5.      j ← π_i                      // 工件编号
6.      o ← k[j]                      // 工序序号
7.      m ← Model.getMachine(j, o)    // 加工机器
8.      p ← Model.getProcessingTime(j, o)  // 加工时间
9.      s ← max(M[m], C[j])           // 最早开始时间
10.     c ← s + p                     // 完工时间
11.     M[m] ← c                      // 更新机器可用时间
12.     C[j] ← c                      // 更新工件完工时间
13.     k[j] ← k[j] + 1               // 推进工序指针
14.     记录 (j, o, m, s, c)
15. END FOR
16. C_max ← max(M[i])                 // Makespan
17. RETURN 调度方案
```

### 4.3 关键公式

**最早开始时间计算：**

$$s_{j,o} = \max\left(M_{\mu_{j,o}},\, C_{j}^{\text{prev}}\right)$$

其中：
- $M_{\mu_{j,o}}$ 为机器 $\mu_{j,o}$ 的当前可用时间
- $C_{j}^{\text{prev}}$ 为工件 $j$ 上一道工序的完工时间（第一道工序时为0）

**完工时间：**

$$c_{j,o} = s_{j,o} + p_{j,o}$$

**状态更新：**

$$M_{\mu_{j,o}} \leftarrow c_{j,o}, \quad C_j \leftarrow c_{j,o}$$

### 4.4 时间复杂度分析

解码器对序列中每个基因执行常数时间操作（使用数组 $C[j]$ 直接读取工件完工时间），总时间复杂度为：

$$T_{\text{decode}} = O(N) = O\left(\sum_{j=0}^{n-1} n_j\right)$$

对于标准JSP（每个工件 $m$ 道工序），$N = n \cdot m$，复杂度为 $O(nm)$。

### 4.5 解的可行性保证

左移贪婪解码保证解的可行性：

1. **工序顺序约束**：通过工件工序指针 $k[j]$ 保证按顺序处理，且使用 $C[j]$ 确保前道工序完成后才开始下道工序
2. **机器独占约束**：通过 $M[m]$ 记录机器可用时间，保证同一机器工序不重叠

## 5 基线算法

### 5.1 FCFS算法

FCFS（First Come First Served）采用严格的串行处理策略，作为最朴素的对比基准。

**算法规则：**
1. 所有工件按编号升序排序（默认0时刻到达）
2. 必须完整加工完当前工件的所有工序后，才开启下一个工件
3. 严格遵循先来先服务原则，不允许并行优化

**特点：** 实现简单但效率低下，工件间无并行性，通常产生较大的Makespan。

### 5.2 SPT算法

SPT（Shortest Processing Time）采用事件驱动的单设备处理逻辑，优先选择加工时间最短的工序。

**算法规则：**
1. 采用全局时间推进 + 事件驱动机制
2. 在每次设备完工事件触发时，收集所有就绪工序（前驱工序已完成）
3. 从就绪工序中选择加工时间最短的工序分配给当前机器
4. 每次只处理一个设备上的任务，避免并发冲突

**选择规则：** 当多个工序就绪时，按 $(p_{j,k}, j, k)$ 升序排序，选择优先级最高（加工时间最短）的工序。

**特点：** 相比FCFS，SPT允许工件间并行加工，通常能获得更小的Makespan，但仍属于贪婪启发式，易陷入局部最优。

| 指标 | 公式 | 含义 |
|------|------|------|
| Makespan | $C_{\max} = \max_{j} c_{j,n_j-1}$ | 最大完工时间 |
| 总完工时间 | $C_{\text{total}} = \sum_j c_{j,n_j-1}$ | 所有工件完工时间之和 |
| 平均流程时间 | $\bar{F} = C_{\text{total}} / n$ | 平均每个工件的流程时间 |
| 机器利用率 | $U = \frac{\sum p_{j,k}}{m \cdot C_{\max}}$ | 机器平均利用率 |
| 总空闲时间 | $I = m \cdot C_{\max} - \sum p_{j,k}$ | 机器总空闲时间 |
| 负载方差 | $\sigma^2 = \frac{1}{m}\sum_m (W_m - \bar{W})^2$ | 机器负载均衡度 |
| 优化率 | $R = \frac{C_{\max}^{\text{base}} - C_{\max}^{\text{GA}}}{C_{\max}^{\text{base}}} \times 100\%$ | 相对基线的改进程度 |

### 6.2 指标解读

- **Makespan**：越小越好，是主要优化目标
- **利用率**：越接近1越好，表示机器闲置少
- **负载方差**：越小越好，表示各机器负载均衡
- **优化率**：正值表示GA优于基线，值越大改进越显著

### 7.4 性能优化效果

针对60×60的大规模JSP实例，通过两项关键优化显著提升性能：

**优化1：解码器O(n²)→O(n)改进**

原实现通过遍历operations列表查找工件前道工序完工时间，时间复杂度 $O(N^2)$。改进为使用数组 $C[j]$ 直接存储工件完工时间，读取复杂度降为 $O(1)$。

**优化2：适应度缓存**

采用哈希表缓存已解码个体的适应度值，避免重复计算。

**性能对比：**

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 单次解码时间 | ~100ms | 1.38ms |
| GA总时间（100代） | 十几分钟 | 6.05秒 |
| 缓存节省解码次数 | - | 26.4% |

|------|----------|---------------------|

GA算法能够找到接近或达到最优解的调度方案，显著优于FCFS和SPT基线算法。

### 7.6 收敛性分析

GA算法的收敛曲线呈现以下特征：
1. **快速下降阶段**：前20-30代Makespan快速下降，算法在解空间中高效探索
2. **缓慢改进阶段**：中期改进速度减缓，算法进行局部精细搜索
3. **收敛阶段**：后期曲线趋于平缓，表明算法已接近最优区域

自适应变异率机制在多样性下降时提升变异率，有助于跳出局部最优，表现为收敛曲线在停滞后的再次下降。

## 8 结论

本文设计并实现了一种基于改进遗传算法的JSP求解系统，主要贡献如下：

1. **算法设计**：采用IPOX交叉算子保持工件间相对顺序，工件块移位变异算子增强局部搜索，结合自适应变异率和精英保留策略，有效平衡了探索与开发

2. **高效解码**：左移贪婪解码器时间复杂度 $O(nm)$，配合适应度缓存机制，使60×60问题求解时间从十几分钟降至6秒

3. **完整评估**：实现了Makespan、利用率、负载方差等多维度KPI指标，以及FCFS和SPT基线对比，为算法性能评估提供全面依据

4. **工程实现**：系统采用模块化设计，核心算法、数据处理、可视化完全解耦，支持标准测试用例和自定义随机生成实例

实验结果表明，本文算法在标准测试用例和大规模实例上均显著优于基线算法，验证了所提方法的有效性和工程实用性。
